## 一、全景概览（一句话总结）

| 论文 | 核心定位 | 一句话总结 |
| :--- | :--- | :--- |
| **Alpha Jungle** (Shi 等, 清华, AAAI 2026) | **数学公式的结构化搜索** | 将 LLM 与 MCTS 结合，用回测的多维度反馈引导 LLM 在受限的数学运算符空间中迭代优化公式型因子。 |

## 三、问题定义（解决的痛点）

两篇论文都旨在解决自动化因子挖掘的效率与可解释性困境，但切入角度不同：

| 维度 | Alpha Jungle | FactorEngine |
| :--- | :--- | :--- |
| **核心痛点** | **搜索效率极低（GP 需数百万次尝试）+ 因子缺乏可解释性（量纲不一致）** | **符号因子表现力受限（无法表达条件分支）+ LLM 逻辑与参数耦合导致算力浪费** |
| **形式化定义** | $\mathcal{F}^* = \arg\max_{\mathcal{F}\subset\mathcal{A}} \mathcal{L}(g^*(\mathcal{F}), \mathbf{Y})$（双层优化） | $\max_{P \in \mathcal{P}} \mathcal{F}(P)$（单层程序优化） |
| **搜索空间本质** | **组合数学空间**（公式树的所有拓扑结构） | **代码语义空间**（所有符合接口规范的 Python 程序） |

## 四、什么是“好因子”？—— 核心定义的分水岭

这是两篇论文最本质的区别所在。

### 4.1 Alpha Jungle：相对百分位排名 + 多维短板约束

Alpha Jungle 采用**动态对抗性评估**，因子得分取决于它在现有优秀因子库 $\mathcal{F}_{zoo}$ 中的相对排位。

$$
R(f, m, \mathcal{F}_{zoo}) = \frac{1}{|\mathcal{F}_{zoo}|} \sum_{f' \in \mathcal{F}_{zoo}} \mathbb{I}(m(f) < m(f'))
$$
$$
e_i(f) = 1 - R(f, m_i, \mathcal{F}_{zoo})
$$

**五个评估维度**：
- **有效性（Effectiveness）**：RankIC
- **稳定性（Stability）**：RankIR
- **换手率（Turnover）**：日换手率（越低越好）
- **多样性（Diversity）**：与库内因子最大相关性
- **过拟合风险（Overfitting Risk）**：LLM-as-a-Judge 主观评分（0~10）

**入库门槛（硬性约束）**：
- RankIC $\ge 0.015$
- RankIR $\ge 0.3$
- 日换手率 $\le 1.6$
- 与库内因子最大相关性 $< 0.8$

**总体得分**：$S(f) = \frac{1}{|\mathcal{D}|} \sum_{i=1}^q e_i(f)$（五维平均）

### 4.2 FactorEngine：绝对加权适应度（Fitness Score）

FactorEngine 采用**绝对统计质量最大化**策略，在验证集上计算加权适应度分数（Fitness Score, FS），作为唯一奖励信号。

$$
FS = \frac{1}{4} \left( IC \times 10 + ICIR + RIC \times 10 + RICIR \right)
$$

| 符号 | 含义 | 权重逻辑 |
| :--- | :--- | :--- |
| **IC** | 预测值与真实收益的线性相关系数 | 乘 10，放大预测精度 |
| **RIC (RankIC)** | 预测排名与真实收益排名的秩相关系数 | 乘 10，放大排序能力 |
| **ICIR** | IC 的均值/标准差（时间稳定性） | 直接加和 |
| **RICIR** | RIC 的均值/标准差（时间稳定性） | 直接加和 |

**辅助软约束**（不进入公式，用于分析验证）：
- 低衰减：年度 IC 趋势平稳
- 高多样性：MDS 投影中离散分布（回转半径 RoG 大）
- 高可执行性：通过语法检查（FE 达 99%）

### 4.3 核心差异对比

| 对比维度 | Alpha Jungle | FactorEngine |
| :--- | :--- | :--- |
| **评分基准** | **相对排位**（在 $\mathcal{F}_{zoo}$ 中的百分位） | **绝对统计量**（固定的 IC/ICIR 加权求和） |
| **评估标准演化** | **动态水涨船高**（库越强，要求越高） | **固定目标最大化**（追求绝对数值增长） |
| **过拟合控制** | LLM-as-a-Judge 主观分 + FSA 结构黑名单 | 样本外长周期盲测 + 严格时序拆分 |
| **优化导向** | **补短板**（选最低维度专项修复） | **总分最大化**（宏观逻辑 + 微观参数协同） |

## 五、核心算法机制对比

### 5.1 Alpha Jungle：LLM + MCTS 四步循环

将因子挖掘重构为**基于树的序贯决策过程**，算法遵循 MCTS 的四阶段循环：

**① 选择（Selection）：基于 UCT 的探索-利用权衡**
$$
a^* = \arg\max_{a \in A(s)} \left( Q(s,a) + c \sqrt{\frac{\ln N_s}{N_{s'}}} \right)
$$
- **创新**：引入**虚拟扩展（Virtual Expansion）**，允许对任意内部节点进行再次优化，而非仅扩展叶节点。

**② 扩展（Expansion）：维度导向的定向修复**
- **短板采样**：通过 Softmax 计算，得分最低的维度（如“换手率太高”）被选中的概率最大：
  $$
  P(i^* = i|s) = \text{Softmax}\left( \frac{e_{\text{max}} \cdot \mathbf{1}_q - \mathbf{E}_s}{T} \right)_i
  $$
- **双阶段 LLM 生成**：先生成**文本化改进建议**（Alpha Portrait），再翻译为具体的**伪代码表达式**。

**③ 评估（Evaluation）：相对百分位排名**
- 因子在各维度上的得分由其在 $\mathcal{F}_{zoo}$ 中的相对排位决定（公式如上）。

**④ 反向传播（Backpropagation）与动态预算**
- $Q(s_k, a_k) \leftarrow \max(Q(s_k, a_k), S(f_{new}))$（只记最高分）
- 若 $S(f_{new})$ 突破记录，搜索预算自动增加 1（动态加时）。

**⑤ 结构多样性调控：频繁子树避免（FSA）**
$$
\text{Support}(\bar{g}) = \frac{1}{|\mathcal{F}_{zoo}|} \sum_{f' \in \mathcal{F}_{zoo}} \mathbb{I}(\bar{g} \subseteq \bar{\mathcal{G}}(f'))
$$
- 支持度排名前 $k$ 的频繁模式被拉入**黑名单**，强制 LLM 避免生成。

---

### 5.2 FactorEngine：宏-微协同进化（Macro-Micro Co-evolution）

将因子进化分解为**逻辑语义探索（宏观）**与**数值参数搜索（微观）**两个完全解耦的层次。

**① 选择（Selection）：基于 UCT 的节点选取**
$$
UCT(v) = Q(v) + c \sqrt{\frac{\ln N_{\text{parent}(v)}}{N_v}}
$$
- 与 Alpha Jungle 类似，但 $Q(v)$ 定义为子树内所有节点的**经验平均分**，而非最大值。

**② 宏观逻辑进化：经验链（CoE）驱动的 LLM 变异**
- **路径评分**：候选路径 $p_i$ 的有效得分减去覆盖度惩罚：
  $$
  S_{total}(p_i) = S_{eff}(p_i) - \gamma \cdot S_{cov}(p_i)
  $$
  $$
  S_{cov}(p_i) = \alpha \frac{|\Phi|}{|p_i|} + \beta \frac{|\Phi|}{|C|}, \quad \Phi = p_i \cap C
  $$
  - 指引 LLM 去学习**“历史上分数高，但与当前路径重合度低”**的进化轨迹。
- **LLM 动作**：输出 `SEARCH/REPLACE` 代码差异（Diff），实现结构性改造，**仅提供参数搜索范围**（如“窗口期 5~30”），不提供具体数值。

**③ 微观参数调优：贝叶斯搜索（完全本地计算，不调用 LLM）**
- **优化目标**：最大化期望提升（Expected Improvement, EI）：
  $$
  EI(\theta) = \int_{-\infty}^{\infty} \max(y^* - y, 0) \cdot p(y|\theta) dy
  $$
- **算法**：TPE（Tree-structured Parzen Estimator）或高斯过程。
- **资源分离**：LLM 完全不参与数值调优，所有参数搜索在本地 CPU 多进程并行执行。

**④ 评估与反馈（Fitness Score）**
- 采用前述 $FS = \frac{1}{4}(IC \times 10 + ICIR + RIC \times 10 + RICIR)$ 作为唯一奖励信号。
- 精英筛选：仅保留评分 > 0.4 且排名前 5 的因子节点，及每个节点下前 10 的参数配置。

**⑤ 反向传播**
- 更新祖先节点的访问计数 $N(v)$ 和平均价值 $Q(v)$，与 Alpha Jungle 的“最大更新”形成对比。

### 5.3 算法机制核心差异汇总

| 维度 | Alpha Jungle | FactorEngine |
| :--- | :--- | :--- |
| **搜索框架** | MCTS（树搜索） | 树结构进化搜索（类似 MCTS，但节点为完整程序） |
| **LLM 职责** | 生成文本建议 + 翻译为伪代码 | 输出代码 Diff（仅宏观逻辑变异） |
| **参数调优** | LLM 在 Prompt 中直接生成具体数值 | **贝叶斯搜索完全接管**（LLM 仅划范围） |
| **变异粒度** | 公式树局部替换 | 代码块级替换（支持条件/循环） |
| **节点价值更新** | **最大值更新**（$\max$） | **平均值更新**（经验平均） |
| **多样性控制** | **FSA**（结构黑名单） | **CoE 覆盖度惩罚** + MDS 散度评估 |
| **搜索预算** | 动态增加（突破记录时 +1） | 固定轮次（200/400 轮） |

## 六、评估指标（共同与各自侧重）

### 6.1 共用指标（预测能力与组合绩效）

| 指标 | 计算方法 | 含义 |
| :--- | :--- | :--- |
| **IC** | 截面皮尔逊相关系数 | 预测精度 |
| **RankIC (RIC)** | 截面斯皮尔曼秩相关系数 | 排序能力（对极端值鲁棒） |
| **ICIR** | $\text{mean}(IC) / \text{std}(IC)$ | IC 的时间稳定性 |
| **RICIR** | $\text{mean}(RIC) / \text{std}(RIC)$ | RIC 的时间稳定性 |
| **年化超额收益 (AER/AR)** | 组合相对基准的超额年化复合收益 | 绝对赚钱能力 |
| **信息比率 (IR)** | $\text{AER} / (\sigma(R_p) \cdot \sqrt{P})$ | 风险调整后收益 |
| **夏普比率 (SR)** | $\sqrt{252} \cdot \text{mean}(r_t) / \text{std}(r_t)$ | 单位总风险的收益 |

### 6.2 各自特色指标

| 特色指标 | 定义 / 计算方法 | 所属论文 |
| :--- | :--- | :--- |
| **搜索效率（有效因子累计数）** | 横轴：生成因子总数；纵轴：有效因子累计数。斜率越大，搜索越高效。 | Alpha Jungle |
| **LLM-as-a-Judge 可解释性排名** | 第三方 LLM 盲审排序（重复 50 次取平均）。 | Alpha Jungle |
| **成本-性能分析（美元/IR）** | 将运行时长与 Token 消耗换算为美元，计算单位成本的信息比率增益。 | Alpha Jungle |
| **可执行比率（Executable Ratio）** | 生成的因子代码通过语法检查并成功运行的比例。 | FactorEngine |
| **回转半径（RoG, Radius of Gyration）** | 基于 MDS 投影的因子空间离散程度量化指标。RoG 越大，因子集多样性越强。 | FactorEngine |
| **调试调用占比（Debug Ratio）** | API 调用中用于修复代码错误的占比。越低说明生成质量越高。 | FactorEngine |

## 八、训练本质与数据驱动

**两篇论文的共性结论**：**均不微调（Fine-tune）LLM**，LLM 仅作为推理引擎（In-context Learning）使用。

| 维度 | Alpha Jungle | FactorEngine |
| :--- | :--- | :--- |
| **LLM 是否被训练？** | **否**，仅作为 API 推理调用。 | **否**，仅作为 API 推理调用。 |
| **真正被优化的对象** | MCTS 树上各节点的 $Q$ 值（历史最高分）和访问计数 $N$ | 搜索树节点的统计量 + 贝叶斯代理模型的概率分布 |
| **核心“训练”框架** | 基于 UCT 的树搜索（无梯度） | 宏 UCT + 微贝叶斯优化（无梯度） |
| **核心奖励信号（环境交互数据）** | 回测产生的多维度绩效指标（IC、换手率、衰减率等） | 验证集上的 $FS = \frac{1}{4}(IC \times 10 + ICIR + RIC \times 10 + RICIR)$ |
| **先验知识数据** | LLM 的预训练权重（内含金融常识） | **多智能体从金融研报（PDF）提取的种子程序** |
| **LLM 上下文数据** | 当前因子 + 维度得分 + $\mathcal{F}_{zoo}$ 中的示例 | **经验链（CoE）**：历史代码、分数变化、失败日志 |
| **为什么不训练 LLM？** | 成本高、灾难性遗忘、即插即用 | 成本高、灾难性遗忘、即插即用 |

## 九、算法更新方向对比

| 维度 | Alpha Jungle | FactorEngine |
| :--- | :--- | :--- |
| **宏观更新方向** | **短板驱动**：通过 Softmax 选中最低分维度（如“换手率高”），LLM 针对该维度生成文本建议与公式修复。 | **经验链驱动**：LLM 综合参考“高绩效但低重叠”的历史路径，输出代码 Diff。 |
| **微观更新方向** | LLM 直接生成具体参数数值（耦合在公式中）。 | **贝叶斯期望提升（EI）**：$EI(\theta) = \int_{-\infty}^{\infty} \max(y^* - y, 0) \cdot p(y|\theta) dy$，概率模型引导参数向高 FS 区域收敛。 |
| **多样性更新方向** | **FSA 黑名单**：高频根基因被禁止生成。 | **覆盖度惩罚（$S_{cov}$）+ MDS 散度评估**：显式鼓励与历史路径低重叠。 |
| **总体指挥棒** | 最大化 $S(f) = \frac{1}{|\mathcal{D}|} \sum e_i(f)$（多维平均分）。 | 最大化 $FS = \frac{1}{4}(IC \times 10 + ICIR + RIC \times 10 + RICIR)$。 |

## 十、终极总结：两篇论文的互补与分野

| 维度 | Alpha Jungle | FactorEngine |
| :--- | :--- | :--- |
| **哲学基础** | **精准符号回归**（在受限空间中求最优公式树） | **图灵完备程序合成**（突破表达力天花板） |
| **对人类知识的利用** | 依赖 LLM 的内在金融常识 + Prompt 示例 | 多智能体**显式提取研报** + 经验链（CoE） |
| **效率优化策略** | 虚拟扩展（深度优先挖掘潜力节点）+ FSA | 宏观/微观分离（LLM 不做参数搜索）+ 贝叶斯并行 |
| **评估动态性** | **相对对抗**（标准水涨船高） | **绝对最大化**（固定加权统计量） |
| **过拟合防御机制** | LLM-as-a-Judge + 样本外测试 | 四阶段严格时序拆分 + 8 年盲测 |
| **代码可执行性** | 数学公式天然可执行 | 显式优化可执行比率（达 99%） |
| **适用场景** | 追求极致可解释性、监管要求严格的公式型因子场景 | 允许复杂程序逻辑、追求超额收益最大化的大规模挖掘场景 |

**一句话总结两者关系**：
> **Alpha Jungle** 是“带着金融直觉的精准狙击手”，在数学公式空间里做极致的定向搜索；**FactorEngine** 是“自带知识库与工具箱的工程队”，用图灵完备的程序语言配合高效自动化调优，系统性铺开挖掘网络。两者代表了自动化因子挖掘从“**表达受限的符号回归**”向“**表达自由的程序合成**”演进的完整光谱。
